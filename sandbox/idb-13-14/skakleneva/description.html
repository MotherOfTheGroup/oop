Tребования • Функциональные требования и требования к реализации
1. Личная страница должна быть разработана с использованием языков HTML, JavaScript и CSS.
2. Личная страница должна содержать одностраничное приложение (single page application, SPA) и визуализируемый тег, требования к которым описаны ниже.
3. Личная страница должна обеспечивать работу пункту 2 функциональных требований(разделы "SPA" и "Визуализируемый тег") в автономном режиме (без подключения к сети Интернет) с помощью стандартного интернет-обозревателя.
4. База данных для хранения настроек цветовой схемы личной страницы должны быть реализована база на json.
5. Авторизация и аутентификация должны быть реазированы с помощью VK API.
6. После авторизации пользователь должен иметь возможность выбрать предпочтительную цветовую схему из предоставленного ему списка (с сохранением настроек).
• Системные требования
Приложение должно занимать не более n мб оперативной памяти устройства;
• Нефункциональные требования
Должны присутствовать ссылки на исходные коды и страницу в wiki.
• Требования к пользовательскому интерфейсу
1. Элементы расположены горизонтально в следующем порядке:
*строка авторизации и аутентификации
*Строка настроек цветовой схемы страницы (видна только авторизированным пользователям)
*Строка ссылок (на исходные коды и wiki-страницу)
*Раздел с требованиями к личной странице
*Раздел с SPA и требованиям к нему
*Раздел с визуализируемым тегом
*Раздел с UML Class diagram
2. Цветовая схема личной страницы состоит из трех цветов (цвет фона страницы, цвет фона блоков и цвет Заголовков) и взяты из следующего списка:
Красный - темно-красный - Рыжий;
Голубой - белый - синий.
